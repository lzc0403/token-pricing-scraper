一、代码质量分析

从可读性、可维护性、性能、最佳实践四个维度评估。

1.1 可读性

  • 语义化缺失：章节标题用 <div class="section-title"> 而非 <h2>，破坏标题层级，影响 SEO 与屏幕阅读器导航。已改 <h2> / <section aria-labelledby>
  • 内联样式滥用：style="overflow-x:auto"、style="position:relative;height:360px" 等散落多处，难以统一维护。已抽为 .table-scroll / .chart-wrap 类
  • 硬编码文案：Hero 副标题 "跨 10 个官方渠道" 与 metrics 卡 "数据源数 8" 数字打架（10 是配置源数含多页 Kimi，8 是 watchlist 实际去重源数）。已改为动态读取实际源数，标签改为「覆盖数据源」
  • 长文本挤压行高：model_raw 含 "qwen3.7-max当前能力等同于…Batch调用半价…折扣" 等长串，表格行被撑得很高。已加 max-width + word-break + title 悬停看全文

1.2 可维护性

  • 生成器与产物耦合正确：site.py 模板化生成，逻辑清晰（_build_site_data → 片段渲染 → 拼装），本次重构保持函数签名不变，main.py 调用零改动。
  • 配置项集中：颜色用 CSS 变量，本次新增 --radius / --space-* / --shadow-3 / --green-deep 体系，间距有据可循。
  • chart 数据结构扩展但向后兼容：chart 行新增 output_rmb / currency 字段，旧 JS 仍可读（取 input_rmb），新 JS 用切换。

1.3 性能

  • 字体加载：已用 preconnect + display=swap，合格。
  • Chart.js CDN：离线时 typeof Chart === 'undefined' 静默跳过，处理得当；但无降级提示。已保留静默跳过（避免误报），图表区在无 Chart 时仍占位不塌陷。
  • Hero mesh blur(8px)：低端设备可能掉帧。已加 @media (prefers-reduced-motion) 关闭浮动动画，并限定动画用 transform（GPU 友好）。
  • 内联 JSON 体量：SITE_DATA 约 8KB 内联，可接受（自包含设计取舍，保证 file:// 可用）。

1.4 最佳实践

  • 缺 SEO meta：无 description / og / theme-color。已补全。
  • 无 skip-link：键盘用户需 Tab 全表。已加「跳到主要内容」。
  • 无打印样式：打印会丢失背景、出现滚动条。已加 @media print。

二、Bug 检测

按严重度 P0 / P1 / P2 分级。P0 为已验证会出错或安全风险；P1 为体验/兼容缺陷；P2 为改进项。

P0
XSS 残留风险（已缓解）：site.py 已对 model_raw 做 _esc() HTML 转义，且 JSON 注入做了 </ → <\/ 防闭合，整体安全。但若未来 scrapers 产出含 <script> 的 model_raw 且忘记走 _esc，仍会注入。本次新增 _esc_attr 用于 title 属性，确保属性上下文也安全；建议在 scrapers 层加 schema 校验。
P1
可访问性：最低价仅靠颜色区分：绿色背景 + 左边框，色盲用户无法识别「最低价」行。已在最低行模型列前置 ★ 图标，多通道冗余。
P1
可访问性：表格无 caption / th 无 scope：屏幕阅读器无法理解表格主题与列关系。已加 <caption>（visually-hidden）与 scope="col"。
P1
可访问性：canvas 无文本替代：图表对读屏器不可见。已加 role="img" + aria-label，并加 aria-live 区域在切换模型时播报数据点数量。
P1
键盘焦点不可见：默认 outline 被浏览器样式覆盖后无明显反馈。已加全局 :focus-visible（2px primary 描边）。
P1
移动端断点单一：仅 760px 一档，小屏（≤480）stat-card 仍 2 列显挤；表格横向滚动无视觉提示。已加 1024 / 760 / 480 三档；表格滚动区 focusable + 自定义滚动条。
P1
图表仅展示输入价：用户无法对比输出价（输出价往往是成本大头，如 Kimi K2.6 输出 27¥ vs 输入 1.1¥）。已加「输入价/输出价」分段切换，tooltip 同时显示输入+输出+币种。
P1
无排序能力：9 张表只能按源字母序，无法按价格排序找最便宜。已实现表头点击排序（模型/源/输入/输出），aria-sort 同步，数值列正确处理空值。
P2
无「贵多少」直观标注：非最低行看不出比最低贵多少。已加 premium 徽标（如 +275%），红色小标签。
P2
无回到顶部：9 张表纵向很长。已加右下角浮动按钮，滚动 600px 后浮现。
P2
表格 hover 无反馈：已加 tbody tr:hover 高亮 + 过渡。
P2
stat-card 无交互：已加 hover 上浮 + 阴影增强。
P2
无 theme-color：移动浏览器地址栏不匹配品牌色。已加 meta theme-color #4434d4。
P2
表格粘性表头缺失：横向滚动时列名滚走。th 已 position:sticky top:0。
P2
图表无动画过渡：切换模型硬切。已加 600ms easeOutQuart 动画。
P2
tooltip 信息单薄：只显示一个 ¥ 值。已扩展为多行：维度值+输入价+输出价+原币种。
P2
图表 y 轴刻度颜色硬编码：已用 #64748d 与表格 mute 统一。
P2
无 OG 标签：分享无预览。已加 og:title / og:description / og:type。

三、UI 美化提升

视觉设计、交互体验、响应式、动画四方面具体改进。

3.1 视觉设计

维度优化前优化后

3.2 交互体验

  • 表格排序：点击表头排序，⇅ 提示可排序，↑↓ 显示当前方向，aria-sort 同步。
  • 图表切换：分段控件切换输入/输出价，aria-pressed 状态同步。
  • hover 反馈：表格行、stat-card、按钮均有过渡。
  • 回到顶部：滚动 600px 浮现，平滑滚动。
  • 键盘可达：skip-link、focus-visible、表格滚动区 focusable。

3.3 响应式

断点优化前优化后

3.4 动画效果

  • Hero mesh 浮动：18s 缓动 alternate，transform 而非 top/left（GPU 友好）。
  • stat-card 上浮：hover translateY(-3px) + 阴影升级，0.25s。
  • 图表过渡：模型/维度切换 600ms easeOutQuart。
  • 回到顶部：opacity + translateY 渐入。
  • 无障碍：prefers-reduced-motion: reduce 自动关闭 mesh 动画。
